Review: Learning Linux Binary Analysis


In less than 300 pages, Ryan “elfmaster” O'Neil provides an approachable introduction to ELF binary hacking.

I am going to come right out and say that I really enjoyed this book. If my memory serves me, Learning Linux Binary Analysis is the first technical book I have ever fully finished. But that accomplishment speaks more to the quality of the book than to my own determination. O'Neil writes with the clear goal of fanning the flames of interest for binary hacking in the reader. Different aspects of ELF Binary hacking are briefly introduced in each of the separate chapters:

  1. The Linux Environment and Its Tools: a basic primer on the Linux environment and its tools. This chapter is skippable if you already are familiar with Linux platform and its binary toolchain.
  2. The ELF Binary Format: an awesome explanation of the essentials of the ELF binary format. The format is a dry subject so the degree to which the book kept my attention in this section was quite impressive. I recommend reading this chapter several times as it took me a few times to have the importance and effect of each field sink in. The rest of the book is based on this section so spending some extra time here is worth it!
  3. Linux Process Tracing: all about how process tracing and ptrace work in Linux. O'Neil also includes a bit on how process tracing can be used for process image reconstruction and code injection. Ptrace anti-debugging is also covered in this chapter.
  4. ELF Virus Technology — Linux/Unix Viruses: the most interesting chapter in the whole book. The “elfmaster” introduces different type of binary infections and the pros and cons of each. However, he doesn't stop there. Step by step algorithms and sample code are actually given for some of the infection methods. O'Neil takes on the guise of a virus writer and explains some of the unique engineering challenges that come with infection engineering. His straightforward explanation of these challenges from a software development standpoint is worth reading the book for.
  5. Linux Binary Protection: following the most interesting chapter with the second most interesting chapter. O'Neil details the different protection schemes that are possible to protect binaries from analysis. There is an excellent portion of this chapter that talks about how a packer/protector like UPX or Shiva works.
  6. ELF Binary Forensics in Linux: a description of the science of binary forensics. I enjoyed this section because it combined Chapters 2 and 4. This chapter is all about training yourself to identify anomalies in the ELF format (Chapter 2) that come about as a result of different types of infection methods (Chapter 4).
  7. Process Memory Forensics: about how to analyze a core dump or memory image of a process. Not much to say about this chapter, wasn't the best, wasn't the worst.
  8. ECFS — Extended Core File Snapshot Technology: a detailed description of ECFS, a system developed by the “elfmaster” himself for enabling core files to hold data relevant to binary forensics. Cool to read about but a bit complicated.
  9. Linux /proc/kcore Analysis: the application of what we have learned to the kernel.

Most of these chapters cover wide fields of binary study. The type of breadth that can generate a whole book on a single topic. However, O'Neil doesn't get caught up in too much minutia of each subject. Even when describing his own work (ECFS or any of the code samples throughout the book), the author makes a point to stay grounded in the topic, while encouraging the reader with external resources if they are interested. I appreciate the effort by the author to describe his work simply.
