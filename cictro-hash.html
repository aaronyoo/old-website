1. Introduction

CictroHash is a challenge from RITSEC CTF 2018. The idea is to use the properties of the hash to find a collision and submit two different strings that cause a collision to a server using a POST request for the flag with the command: 

curl -X POST http://fun.ritsec.club:8003/checkCollision \
--header "Content-Type: application/json" \
--data '{"str1": "{{INSERT_STR1}}", "str2": "{{INSERT_STR2}}"}'

2. Solution

The solution was actually pretty simple once I understood the cipher and how to implement it. I had a bit of trouble implementing with Python3 due to my lack of practice but I eventually got a working implementation of the cipher: 

def pad(txt):
  while (len(txt) % 4 != 0):
      txt += '\0'
  return txt

def create_w(s):
    w = [[s[0], s[1], s[2], s[3]],
         [s[4], s[5], s[6], s[7]]]
    return w

def f(S):
    w = create_w(S)
    for _ in range(50):
        triangle(w)
    return [w[0][0], w[0][1], w[0][2], w[0][3], w[1][0], w[1][1], w[1][2], w[1][3]]

def triangle(w):
    a(w)
    b(w)
    c(w)
    d(w)

def a(w):
    w[0], w[1] = w[1], w[0]

def b(w):
    w[0][0] ^= w[1][3]
    w[0][1] ^= w[1][2]
    w[0][2] ^= w[1][1]
    w[0][3] ^= w[1][0]

def c(w):
    t = [[w[1][3], w[1][0], w[1][2], w[0][0]],
            [w[1][1], w[0][3], w[0][1], w[0][2]]]
    w[0][0] = t[0][0]
    w[0][1] = t[0][1]
    w[0][2] = t[0][2]
    w[0][3] = t[0][3]
    w[1][0] = t[1][0]
    w[1][1] = t[1][1]
    w[1][2] = t[1][2]
    w[1][3] = t[1][3]


def d(w):
    def rotate_right(n):
        return ((n >> 1) | (n << 7)) & 0b11111111
    def rotate_left(n):
        return ((n << 1) | (n >> 7)) & 0b11111111

    w[0][0] = rotate_left(w[0][0])
    w[1][0] = rotate_left(w[1][0])
    w[0][2] = rotate_left(w[0][2])
    w[1][2] = rotate_left(w[1][2])
    
    w[0][1] = rotate_right(w[0][1])
    w[1][1] = rotate_right(w[1][1])
    w[0][3] = rotate_right(w[0][3])
    w[1][3] = rotate_right(w[1][3])

def hash(pre_image):
    S = [31, 56, 156, 167, 38, 240, 174, 248]
    pre = pre_image
    pre = pad(pre)
    for i in range(0, len(pre), 4):
        pre_block = pre[i:i+4]
        for j in range(4):
            S[j] ^= ord(pre_block[j])
        S = f(S)

    answer = ""
    for i in range(4):
        answer += "{0:x}".format(S[i])
    return answer

One mistake that I made was that I didn't do all the assignments in the gamma function simultaneously at first. Instead, I did them sequentially which caused me to emulate the wrong behavior. I knew I had to get a working implementation of the cipher first so I could check any work I did before submitting to the server. Otherwise I would be essentially just guessing. But after I had a working implementation, it was just a simple birthday attack using a hashmap to find several hundred collisions in a few seconds. Here is code for the birthday attack (that works in conjunction with the code above): 

birthday_dict = {}
while (True):
    pre = "".join(random.choices(string.ascii_letters + string.digits, k = random.randint(1, 5))) 
    hash_val = hash(pre)
    if (hash_val in birthday_dict and birthday_dict[hash_val] != pre):
        print(pre, birthday_dict[hash_val])
    else:
        birthday_dict[hash_val] = pre

Running my script, I found that 3w and d result in a collision. I submitted these strings to the server using the POST request format shown at the beginning of this article and got the response: 

{
  "flag": "RITSEC{I_am_th3_gr3@t3st_h@XOR_3v@}",
  "success": true
}

The flag is: RITSEC{I_am_th3_gr3@t3st_h@XOR_3v@}.

3. Sponge Construction

This challenge is centered around what is called a Sponge Construction (or Sponge Function). In the context of cryptography, a Sponge Construction is a style of hash that:

  1. Takes variable length input and produces variable length output
  2. Has an absorbing phase and a squeezing phase

Note: The portion of state memory never gets XOR'd with anything. The only transformation that ever gets applied to is function . Despite this fact, the memory region is extremely important because it provides resistance to preimage attacks. If there was no C region, an attacker would easily be able to reverse the hash by taking and reapplying XOR to the region at each iteration.

Two different types of attacks in which the level of resistance is determined by the size of the region:

  1. Preimage attack: an attack where you start with a hash value and are able to derive a preimage that hashes to that value.
  2. Collision attack: an attack where you are able to easily (computationaly feasibly) find two different preimages that hash to the same value.

The CictroHash function was an example of a simplified Sponge Function, and although you technically didn't need this knowledge to solve the problem I enjoyed learning about the Sponge Construction overall.

4. Birthday Attack

In order to complete the challenge I had to learn about the birthday attack. This attack relies on the probability behind the birthday problem, which is essentially a small bit of counterintutive probability theory. Basically, in order to conduct a birthday attack (a type of collision attack) we need to just keep hashing random values until we find a collision. By the birthday problem, this actually doesn't take as long as we think it will take. This is how I found the collision of 3w and d earlier and solved the challenge.
