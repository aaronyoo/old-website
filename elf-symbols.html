1. ELF Symbols

Shared objects and dynamically linked executables often have 2 symbol tables. One .symtab (symbol table) table and one .dynsym (dynamic symbol) table. The functions of the tables are as follows:

  • .dynsym: Contains only the global symbols needed at runtime.
  • .symtab: Contains all symbols, including the ones in .dynsym

So .dynsym is a subset of .symtab. The rationale for this is that .dynsym is necessary for dynamic linking at runtime and .symtab is not. Thus, .dynsym is mapped into memory at runtime but .symtab is not. When using the strip utility, to strip the symbols from a binary, the .symtab table is destroyed while the .dynsym table stays intact since it is needed at runtime.

It is important to note that the .dynsym table is only needed for dynamically linked executables. If you statically link your executable, then there will only be a .symtab table (more on this later).

2. An ELF Parser

I finished taking notes on the first few chapters of Learning Linux Binary Analysis but felt like I didn't quite have a good grasp on the layout of an ELF binary overall. In specific, the concept of program headers vs. sections headers was still itching my brain. Because of this bit of confusion, I decided to write a simple ELF parser in order to understand the format better.

2.1. Requirements

I wanted to start off with some concrete requirements for my parser. Before I started coding I used output from the readelf command to figure out what exactly I wanted my parser to spit out. For me, it was a fine balance between doing enough parsing that I understood the format well and not reimplementing the entirety of readelf. Thus, I came up with this tenative list of information I wanted my ELF parser to output.

  • Magic Bytes
  • Entry Point Address
  • Start of Program Headers
  • Start of Section Headers
  • Size of ELF Header
  • Size of Program Headers
  • Number of Program Headers
  • Number of Section Headers
  • All the Section Headers with their appropriate names and offsets
  • All the Program Headers with their appropriate types, offsets, and virtual addresses

2.2. Takeaways

There were a few things I picked up while programming the parser:

Program Headers vs. Section Headers: As stated before, program headers and section headers were confusing to me. In retrospect, I think it is because the way the book describes the relationship between these (not to knock the book). It probably assumed I knew more about linking than I actually did. The book makes the clear point that section headers are not needed at runtime and can be stripped from the binary. However, from this point I wasn't able to visualize the “different views” of linking and execution. At link time, the section headers are used and the program headers are not needed. At execution time, the program headers are used and the section headers are not needed. Here is a diagram that really helped me:

Mmap: I used mmap for the first time when writing my parser. Initially I thought about using malloc to allocate appropriately sized buffers and then reading the header and tables in from a file pointer. However, I decided that mmap would suit my needs better because it would allow me fast random access to the entire file instead of using an lseek file descriptor. I used mmap to map the entire ELF binary into the memory space of my process and then just used pointers to access the data inside.

2.3. Code

#include <stdint.h>
#include <stdlib.h>
#include <elf.h>
#include <stdio.h>
#include <string.h>
#include <sys/mman.h>

void readElfHeader(Elf64_Ehdr *ehdr);

int main(int argc, char **argv) {
    if (argc != 2) {
        printf("Incorrect number of arguments\n");
        exit(0);
    }

    /* Open the file */
    FILE *fp = fopen(argv[1], "rb");
    if (fp == NULL) {
        perror("fopen");
        exit(-1);
    }

    /* Get the size of the file */
    fseek(fp, 0L, SEEK_END);
    long int file_size = ftell(fp);
    rewind(fp);

    /* Map the entire binary into memory for quick random access */
    uint8_t *addr = mmap(NULL, file_size, PROT_READ, MAP_PRIVATE, fileno(fp), 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        exit(-1);
    }

    /* Assign ELF Header, program header table, and section header table */
    Elf64_Ehdr *ehdr = (Elf64_Ehdr *)(addr);
    Elf64_Phdr *phdr = (Elf64_Phdr *)(addr + ehdr->e_phoff);
    Elf64_Shdr *shdr = (Elf64_Shdr *)(addr + ehdr->e_shoff);

    /* Make sure it is a valid ELF Binary by checking magic numbers */
    if (ehdr->e_ident[0] != 0x7f ||
        ehdr->e_ident[1] != 'E' ||
        ehdr->e_ident[2] != 'L' ||
        ehdr->e_ident[3] != 'F') {
            printf("File is not an ELF Binary\n");
            exit(-1);
    }

    /* Make sure it is an executable ELF file */
    if (ehdr->e_type != ET_EXEC && ehdr->e_type != ET_DYN) {
        printf("%d\n", ehdr->e_type);
        printf("ELF file is not executable\n");
        exit(-1);
    }

    /* Extract Information from the ELF Header */
    readElfHeader(ehdr);

    /* Get the string table */
    uint8_t *string_table = &addr[shdr[ehdr->e_shstrndx].sh_offset];

    /* Print Section Header Table */
    printf("Section Headers:\n");
    printf("\tName : Offset\n");
    for (int i = 0; i < ehdr->e_shnum; i++) {
        printf("\t%s : 0x%lx\n", &string_table[shdr[i].sh_name], shdr[i].sh_offset);
    }
    printf("\n");

    /* Print Program Header Table */
    printf("Program Headers:\n");
    printf("\tType : Offset : Virtual Address\n");
    for (int i = 0; i < ehdr->e_phnum; i++) {
        printf("\t%d : 0x%lx : 0x%lx\n", phdr[i].p_type, phdr[i].p_offset, phdr[i].p_vaddr);
    }
    printf("\n");
}

/*
 * Extract information from the ELF Header:
 *         Magic Bytes
 *         Entry Point Address
 *         Start of Program Headers (offset)void readSectionHeaders(Elf64_Shdr *shdr, Elf64_Shdr *string_table)
 *         Start of Section Headers (address)
 *         Size of ELF Header
 *         Number of Program Headers
 *         Number of Section Headers
 */
void readElfHeader(Elf64_Ehdr *ehdr) {
    printf("ELF Header (%d bytes):\n", ehdr->e_ehsize);
    
    /* Print Magic Bytes */
    printf("\tMagic:\t");
    for (int i = 0; i < 16; i++) {
        printf("%02x ", *(((uint8_t *) ehdr) + i));
    }
    printf("\n");

    /* Print Entry Point Address */
    printf("\tEntry Point:\t%lx\n", ehdr->e_entry);

    /* Print Start of Program Headers */
    printf("\tStart of Program Headers:\t%ld (bytes into file)\n",
            ehdr->e_phoff);

    /* Print Start of Section Headers */
    printf("\tStart of Section Headers:\t%ld (bytes into file)\n",
            ehdr->e_shoff);

    /* Print Size of Program Headers */
    printf("\tSize of Program Headers:\t%d (bytes)\n", ehdr->e_phentsize);

    /* Print Number of Program Headers */
    printf("\tNumber of Program Headers:\t%d\n", ehdr->e_phnum);

    /* Print Size of Section Headers */
    printf("\tSize of Section Headers:\t%d (bytes)\n", ehdr->e_shentsize);

    /* Print Number of Section Headers */
    printf("\tNumber of Section Headers:\t%d\n", ehdr->e_shnum);

    printf("\n");
}
