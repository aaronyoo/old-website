1. Introduction

1.1. Background

One of the more recent advances in network verification, introduced by Microsoft Research and Plotkin, is using graph reduction to speed up the resolution of reachability queries between nodes[3]. In this case, the edges of the graph are the links between network nodes and performing reduction based on equivalence classes simplifies the network topology, making future operations on this topology easier. Plotkin's network graph reduction uses the disjoint-set (Union-Find) data structure as a way to determine membership of an equivalence class in linear time. But the Union-Find structure itself isn't too novel and was introduced by Galler in 1964[1]. The application of this algorithm to graph reduction followed soon after.

1.2. Problem

In light of my recent exposure to Plotkin's work through my Network Verification class, I wanted to further explore the Union-Find data structure and it's application to reduction problems. Union-Find is applicable to problems that exhibit symmetry. Take Plotkin's network reduction as an example, the routers in the network exhibit a lot of redundancy (and thus symmetry) in their routing rules and locations in the network topology. This symmetry allows the establishment of equivalence classes. But at a more general level, this symmetry is caused by consistent design patterns. There are many other fields where design patterns are consistent in this way. One field where this is possibly applicable is reverse engineering. Since compilers output code following mostly predictable patterns, it may be possible to perform dead code elimination or deobfuscate obfuscated paths automatically. In summary, this tutorial aims to:

  1. Provide an intuition about the Union Find Disjoint Set data structure by introducing its operations and theoretical background, then solving several classical problems.
  2. Explore the possible application of graph reduction and Union-Find to binaries using Binary Ninja's Intermediate Language.

2. Union Find

2.1. Description

Union-Find Disjoint Set (UFDS) is a data structure that models disjoint sets. These sets are sometimes interpreted as equivalence classes. The nice thing about UFDS is that it is able to determine which set an item belongs to in amortized time. UFDS can be applied to any problem that needs to model disjoint equivalence classes and test for membership quickly. One classical problem that can be solved with UFDS is finding connected components in a graph.

Note that testing for membership is not supported efficiently by other data structures. The most similar structure in C++ STL is a set, which does not support efficient testing for membership of disjoint sets. A vector of sets provides the same functionality in time by iterating through the vector and checking for membership in each of the sets which is typically an operation.

UFDS is implemented by keeping track of the parent of each item. This can be done simply by an array. If the parent of the element is the element itself, then the element is a representative item (the item that represents the equivalence class). In other words, the element is a representative element if and only if parent(element) == element. If this is not true, then we need to recursively visit the parent until the condition holds true to obtain the representative element. The representative elements are the roots of trees in which the leaves are elements in the same class as the representative element. Put simply, each disjoint set is a tree and UFDS is a forest of these trees.

Lastly, there is the notion of rank, an upper bound on the distance of a node away from the root (representative). The main function of rank is to provide a heuristic used when taking the union of two sets. When taking the union of two sets (trees) it is best to keep them as short as possible .
